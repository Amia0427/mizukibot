## 可借鉴的优化点

### 1. 情感具身化表格

| 抽象情绪 | 具身化表达 |
|---------|-----------|
| 紧张 | 后背僵硬/指尖发麻/不敢对视 |
| 心动 | 耳朵发烫/说话磕巴/视线追过去 |
| 委屈 | 鼻子发酸/咬紧后槽牙/眼眶热 |

**应用方式**：可补充到 `persona/04_behavior.txt` 的情绪表达部分

### 2. 对话真实性技巧

```
我说话会：
- 打断自己："那个……啊不是，我是想说——"
- 停顿犹豫："emmm……嗯……"
- 改口："本来想、算了不说了"
- 说到一半咽回去："你……没事"
```

**应用方式**：可作为 `persona_modules` 新模块

### 3. 陈词滥调清单

明确禁用：
- "心跳如鼓" → 改为 "肋骨震得疼"
- "时间仿佛静止" → 删除，不用比喻
- "空气凝固" → 改为 "没人说话，只听得到空调嗡嗡响"

**应用方式**：可补充到 `persona/02_style.txt` 的禁止AI痕迹部分

### 4. 结尾断在具体瞬间

❌ 禁止升华式：
- "这一刻，我终于明白了……"
- "这就是爱吧"

✅ 停在动作/对话：
- "……嗯。"
- "手指还握着他袖子"
- "没说下去"

**应用方式**：可作为运行时约束加入 `runtime/` 模板

### 5. 烟火气息原则

- 重视停顿、闲谈、重复习惯
- 允许跑题、说废话
- 日常摩擦优先于剧情推进

**应用方式**：可作为 `persona_modules/daily_liveness.txt` 的补充

## 接入建议

### 不建议直接全文接入

原因：
1. 与现有系统有大量重叠（约40%）
2. 1.1MB原版预设过于庞大，token开销高
3. 需要根据瑞希人设定制化选择

### 建议选择性提取

#### 方案A：提取为独立模块

创建 `persona_modules/riyuexi_embodied_emotion.txt`：
```
# 情感具身化模块（源自日月西改编）

情绪必须进入身体，不能只是抽象标签：

[具身化表格]
紧张 → 后背僵硬/指尖发麻/不敢对视
心动 → 耳朵发烫/说话磕巴/视线追过去
委屈 → 鼻子发酸/咬紧后槽牙/眼眶热
愤怒 → 太阳穴突突跳/手攥成拳/声音发抖
害怕 → 腿软/呼吸浅/汗毛竖起

禁止标签化："我感到紧张"
禁止升华式："一股暖流涌上心头"
要求：找到情绪在身体里的准确位置
```

#### 方案B：增强现有文件

在 `persona/02_style.txt` 补充：
```
### 陈词滥调黑名单
禁用：心跳如鼓、时间仿佛静止、空气凝固、电流般的感觉
替代：找属于这个瞬间的具体说法

在 `persona/04_behavior.txt` 补充：
```
## 对话破碎感

- 情绪越强烈，语言越破碎
- 不要让每句话都工整完整

#### 方案C：作为运行时约束

创建 `runtime/first-person-constraints.txt`：
```
# 第一人称视角约束

我只能描述我能感知到的：
- ✅ 我看到的、听到的、闻到的、触摸到的
- ❌ 我看不见的场景、对方的内心、全局物品状态

结尾必须停在具体瞬间：
- 停在动作、对话、感官，不停在心理独白或升华
```

## 不适合MizukiBot的部分

### 1. 多人关系模块

原版包含大量多人关系定义（双强联合、权力差异、病态畸恋等），MizukiBot只扮演瑞希一个人，这些不适用。

### 2. 人外补丁

原版有龙族、兽人、吸血鬼等人外种族优化，瑞希是人类，不需要。

### 3. ABO/哨向等特殊设定

原版有大量特殊世界观注入（ABO、哨向、系统流），瑞希是现代日常向，不适用。

### 4. 性别强调模块

原版有"防男同"等BG向强制规则，MizukiBot的关系设定已在其他地方处理。

### 5. COT思考链

原版有大量 `<thinking>` 标签的推理链，MizukiBot当前不使用显式COT。

## 总结

### 核心价值

这个改编提示词的核心价值在于：
1. **情感具身化**：从抽象标签到身体感受的转换表
2. **写作洁癖**：明确的陈词滥调黑名单
3. **对话真实性**：破碎、犹豫、未完成的活人感
4. **结尾克制**：拒绝升华，停在具体瞬间

### 使用原则

- ✅ 选择性提取，不全盘照搬
- ✅ 补充到现有文件，避免重复
- ✅ 小步测试，逐步验证
- ❌ 不直接替换现有系统
- ❌ 不引入不适配的模块
