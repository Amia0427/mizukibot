# QQ Action Routing

更新时间：2026-06-03 08:24 +08:00

## 定时消息误判修复

2026-05-25 00:30 +08:00 的群聊事故中，用户本轮只是回复道歉，但增强后的 `effectiveIntentText` 带有引用内容：

- 引用文本包含“没错到点上吧”，命中旧的 `到点` 定时信号。
- 引用包装包含“引用消息”，命中旧的群消息目标词。
- 路由先进入 `qq_schedule_message`，planner 又返回 `chat_only`，最终执行层因无工具步骤回了当时的定时消息不可用兜底。

修复后，`core/router/index.js` 对 QQ action 这类写操作只使用本轮 CQ 清洗后的 `currentTurnText` 判定，不再用带引用摘要的增强文本触发写操作。直接聊天、记忆检索和引用追问仍保留增强上下文。

2026-06-03 08:24 +08:00 起，定时消息不可用兜底改为瑞希口吻：“定时消息这边刚刚没接住。把时间说得更清楚一点，我再试一次。”。

同时收紧 `到点`：只有和“发/发送/提醒/通知/叫我/跟大家说”等动作语境共现时才算定时信号；`定时`、`稍后`、明确日期时间、N 分钟/小时后、每天/每周时间和 cron 仍保持原支持。

回归测试：

- `tests/routerChineseKeywords.test.js` 覆盖事故原文不再进入 `qq_schedule_message`。
- 同文件覆盖“到点提醒本群该睡觉了”仍进入定时群消息路由。
