## 接入内容

### 1. 新增模块

#### `persona_modules/embodied_emotion.txt`
**功能**：情感具身化对照表  
**优化点**：
- 将抽象情绪转换为身体反应（紧张→后背僵硬/指尖发麻，心动→耳朵发烫/说话磕巴）
- 禁止标签化表达（"我感到紧张"）
- 要求身体反应先于意识判断
- 多感官叠加表达

**配置**：
- `priority`: 615
- `budget_tokens`: 350
- `conflict_tags`: []
- `stage`: main

#### `persona_modules/dialogue_fragments.txt`
**功能**：对话破碎感  
**优化点**：
- 打断自己、停顿犹豫、改口、说到一半咽回去
- 情绪越强语言越破碎的分级使用
- 口语化标记（emmm/啊/嗯/诶/唉）
- 与情感具身化叠加使用

**配置**：
- `priority`: 616
- `budget_tokens`: 320
- `conflict_tags`: []
- `stage`: main

### 2. 增强现有文件

#### `persona/02_style.txt`
**新增部分**：陈词滥调黑名单

**禁用表达**：
- 心跳如鼓/心脏漏跳一拍 → 改为"肋骨震得疼"/"心跳乱了"
- 时间仿佛静止 → 删除这种比喻
- 空气凝固 → 改为具体描述
- 电流般的感觉 → 改为"触电似的缩回手"
- 眼神深邃/笑容意味深长 → 删除模糊形容

**禁用升华式**：
- "这一刻，我终于明白了……"
- "这就是爱吧"
- "心底某个角落悄悄松动了"
- "仿佛命运的齿轮开始转动"
- "一股暖流涌上心头"

**替代原则**：用具体的动作、感官、身体反应替代抽象比喻

### 3. 配置更新

#### `prompt-manifest.json`
注册两个新模块到 persona_module 系统

## 改编说明

### 改编来源
【日月西】Gemini & Claude v0.1 _电波系.json（1.1MB 第三人称小说生成预设）

### 转换方向
第三人称小说叙事 → 第一人称角色扮演

### 保留的优化点
✅ 写作洁癖：拒绝陈词滥调  
✅ 情感具身化：情绪进入身体  
✅ 对话真实性：破碎、犹豫、打断  
✅ 因果重量：选择有代价  
✅ 烟火气息：日常摩擦优先  

### 删除的不适配部分
❌ "月读/天照"创造者视角  
❌ "你是小说家"身份定位  
❌ 全局物品追踪  
❌ 多角色群像塑造  
❌ 破限和NSFW专项规则  
❌ 第三人称叙事框架  

### 新增的第一人称约束
🆕 认知有限：允许误解、错过、不确定  
🆕 视角约束：只能描述看到/听到/感受到的  
🆕 身体体验：从"观察角色"改为"我的感受"

## 模块激活机制

### embodied_emotion
**自动激活场景**：
- 情绪波动明显的对话
- 关系发生微妙变化
- 尴尬、紧张、心动等强烈情绪

**不激活**：平静闲聊可以不带身体反应

### dialogue_fragments
**激活程度**：
- 平静时：70%完整 + 30%破碎
- 情绪波动：50%完整 + 50%破碎
- 强烈情绪：30%完整 + 70%破碎

**适合场景**：
- 尴尬场景
- 紧张对话
- 情感表达
- 被突然问到敏感问题

## 未来优化建议

### 阶段性测试
1. 观察情感具身化是否改善情绪表达的具体性
2. 检查对话破碎感是否与瑞希风格兼容
3. 监控是否出现过度破碎或过度具身化

### 可选的进一步接入
如果效果良好，可继续从参考文件提取：
- 烟火气息细节（日常摩擦、未说出口的话）
- 结尾克制约束（停在具体瞬间，不升华）
- 第一人称视角约束（认知有限、视角限制）

### 不建议接入的部分
- 多人关系定义（瑞希只扮演一个人）
- 人外补丁（瑞希是人类）
- ABO/哨向等特殊设定（不适配世界观）
- COT思考链（当前不使用显式COT）
